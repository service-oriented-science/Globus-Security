Test Metrics :

The test count is an indicator of the size of the system. Test coverage is a percentage measuring the fraction of code that has been changed. If you divide the test coverage by the test count, you get test coverage per test. This measure should not be increasing for every version, since that indicates a larger scope for the test. Test coverage on changed methods measures the amount of tests one has for the code that is maintained.
